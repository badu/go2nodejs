  • Conclusion

    green language
  • performance
  • open source
  • simple
  • simple concurrency
  • cloud native
  • Go has performance. In one video comparing Go, Rust and C++, someone said that it is being impressed with the resilience and consistence of Go.
  • Go is open-source. With each release the speed is increased and carefully chosen language features are being introduced. I can still run code I wrote 7 years ago, which I cannot say about Java, which introduced some breaking changes in the tooling (Maven, in my case)
  • In its simplicity, Go is - as a Go proverb says - `easy to learn, hard to master`. In day to day jobs, a developer would rarely need to master it (optimization jobs are pretty rare and once you've solved the problem is solved for good).
  • The concurrency is super-duper easy : any junior can pick it up. Indeed, we should favor seniors, but from my experience juniors adopt the language with ease and pleasure, when they are facing good mentorship and right explanations.
  • Last but not least, Go is cloud-native - for real - not just a fancy word. Both Docker and Kubernetes are written in Go language and a developer can pick up DevOps related knowhow just by looking through code.
